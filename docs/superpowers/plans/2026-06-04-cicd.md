# CI Pipeline Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a GitHub Actions CI pipeline that runs backend tests and frontend build in parallel on every push to `main`.

**Architecture:** A single workflow file at `.github/workflows/ci.yml` defines two parallel jobs — `backend` (Maven tests) and `frontend` (Vite build). Both run on `ubuntu-latest` GitHub-hosted runners. No real services are needed because the backend tests use Mockito mocks.

**Tech Stack:** GitHub Actions, Java 21 (Temurin), Maven, Node 20, npm

## Task 1: Create the GitHub Actions workflow

**Files:**
- Create: `.github/workflows/ci.yml`

- [ ] **Step 1: Create the workflow file**

Create the file `.github/workflows/ci.yml` with the following content:

```yaml
name: CI

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  backend:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Set up Java 21
        uses: actions/setup-java@v4
        with:
          java-version: '21'
          distribution: 'temurin'
          cache: 'maven'

      - name: Run tests
        run: mvn -B test --no-transfer-progress

  frontend:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Set up Node 20
        uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'npm'
          cache-dependency-path: frontend/package-lock.json

      - name: Install dependencies
        working-directory: frontend
        run: npm ci

      - name: Build
        working-directory: frontend
        run: npm run build
```

- [ ] **Step 2: Commit**

```bash
git add .github/workflows/ci.yml
git commit -m "ci: add GitHub Actions CI pipeline"
```

A multi-tenant document generation and notification platform.

## Task 3: Push and verify

- [ ] **Step 1: Push to GitHub**

```bash

```

- [ ] **Step 2: Watch the pipeline run**

Go to `https://github.com/averyhinazuki/DocPlatform/actions`

You should see a workflow run appear within seconds. Click into it to watch both jobs (`backend` and `frontend`) run in parallel. Each job shows individual step logs.

Expected: both jobs show a green ✓. Total time should be under 3 minutes.

- [ ] **Step 3: Verify the badge on the repo page**

Go to `https://github.com/averyhinazuki/DocPlatform`

The README renders at the bottom of the page. The badge should display `CI | passing` in green.

If either job fails, click the failing job → find the failing step → read the log. Common causes:
- `backend` fails: a test that was passing locally is failing in CI — check the error message in the Maven log
- `frontend` fails: `npm ci` fails if `package-lock.json` is out of sync with `package.json` — run `npm install` locally and commit the updated lock file
